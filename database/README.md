# Database Component

This directory contains the SQLite database for the AgenKampus educational demo.

## Files

- **schema.sql**: Database schema with table definitions and dummy data
- **setup_database.py**: Python script to create and initialize the database
- **kampus.db**: SQLite database file (generated by setup script)

## Database Schema

### Tables

1. **dosen** (Lecturers/Academic Advisors)
   - `id`: Primary key
   - `nama_dosen`: Lecturer name
   - `nidn`: National lecturer ID number (unique)

2. **mahasiswa** (Students)
   - `id`: Primary key
   - `nama_mahasiswa`: Student name
   - `nim`: Student ID number (unique)
   - `id_dospem`: Foreign key to dosen (academic advisor)

3. **mata_kuliah** (Courses)
   - `id`: Primary key
   - `nama_matkul`: Course name
   - `sks`: Credit hours

4. **transkrip** (Transcript)
   - `id_mahasiswa`: Foreign key to mahasiswa
   - `id_matkul`: Foreign key to mata_kuliah
   - `nilai_huruf`: Letter grade (A, B, C, etc.)

## Setup

Run the setup script to create the database:

```bash
python setup_database.py
```

This will:
1. Remove any existing `kampus.db` file
2. Create a fresh database
3. Execute the schema.sql file
4. Verify table creation and data insertion
5. Run a test query

## Verification

After setup, you can explore the database using:

```bash
# SQLite command line
sqlite3 kampus.db

# Example queries
SELECT * FROM dosen;
SELECT * FROM mahasiswa;

# Test the JOIN query used in demo
SELECT m.nama_mahasiswa, d.nama_dosen
FROM mahasiswa m
JOIN dosen d ON m.id_dospem = d.id;
```

## Educational Notes

This database demonstrates:
- **Relational design** with foreign keys
- **Many-to-many relationships** (transkrip links students and courses)
- **JOIN queries** for retrieving related data
- **Data integrity** with NOT NULL and UNIQUE constraints

The database is intentionally kept simple for educational purposes but represents realistic academic data structures.
